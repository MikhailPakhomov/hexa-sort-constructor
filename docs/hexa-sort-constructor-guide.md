# Hexa Sort — краткая инструкция по конструктору уровней

## Быстрый сценарий работы

1. Укажите **ID уровня** и **Название**.
2. Настройте цель и размер поля.
3. Кликами сформируйте игровое поле.
4. Создайте пачки и перетащите нужные пачки на поле.
5. Настройте жёсткую и случайную очереди.
6. Нажмите **▶ Тестировать** и пройдите уровень.
7. Нажмите **Сохранить уровень**, затем выгрузите JSON или общий ZIP.

## Верхняя панель

- **Скачать ZIP** — скачивает архив со всеми сохранёнными уровнями. Каждый уровень находится в отдельном JSON-файле. Несохранённые изменения в архив не попадают.
- **Сохранённые уровни** — показывает сохранённые конфиги и загружает выбранный уровень в редактор.
- **Сохранить уровень** — сохраняет текущий уровень в браузере. При совпадении названия редактор предложит обновить существующий уровень.
- **Копировать JSON** — копирует готовый конфиг текущего состояния в буфер обмена.
- **Скачать JSON** — скачивает только текущий открытый уровень. Предварительно сохранять его необязательно.

> Сохранённые уровни находятся в локальном хранилище браузера. Они привязаны к этому браузеру и компьютеру; очистка данных сайта может удалить их. Для резервной копии используйте ZIP.

## Настройки уровня

### ID и название

- **ID уровня** — технический уникальный идентификатор и имя JSON-файла. Рекомендуемый формат: `level-01`, `level-02` и т. п.
- **Название** — отображаемое имя уровня.

### Цель

- **Любой цвет** — в зачёт идут собранные гексы любого цвета.
- **Конкретный** — в зачёт идёт только выбранный цвет.
- **Количество** — сколько гексов нужно собрать для победы.

### Настройки поля

- **Столбцы / Строки** задают габариты сетки. Диапазон: 2–12 столбцов и 2–16 строк.
- Клик по серой ячейке делает её активной.
- Клик по активной ячейке выключает её. Если на ней стояла пачка, пачка также удаляется с поля.
- Белые ячейки доступны в игре, серые — отсутствуют.

## Игровое поле

- Перетащите пачку из **Библиотеки пачек** на активную ячейку, чтобы сделать её стартовой пачкой на поле.
- Повторное перетаскивание заменяет пачку в ячейке.
- Нажмите значок **🔒** на пачке, чтобы поставить или снять блокировку.
- Правый клик по пачке открывает дополнительные действия: изменить число гексов для снятия замка или убрать пачку с поля.
- Счётчики справа показывают количество активных слотов и установленных пачек.
- **▶ Тестировать** запускает текущую версию уровня без обязательного сохранения.

## Библиотека пачек

Пачка — это стопка гексов, которая выдаётся игроку или заранее стоит на поле.

- **+** создаёт новую пачку.
- Нажатие **✎** открывает редактирование существующей пачки.
- В редакторе пачки нажимайте на цвета или перетаскивайте их в стопку.
- Порядок состава читается как **низ → верх**: первый элемент находится снизу, последний — сверху.
- Элементы можно перетаскивать, двигать стрелками и удалять.
- Карточку готовой пачки можно перетащить на активную ячейку поля.

## Очереди

### Жёсткая очередь

- Определяет пачки в заданном порядке, максимум — 9 элементов.
- Первые 3 элемента отображаются под полем и образуют стартовую руку игрока.
- Элементы с 4-го по 9-й выдаются далее по очереди.
- В каждом слоте можно выбрать конкретную пачку или **Random по весам**.
- Кнопка **×** удаляет слот; **+ Добавить в очередь** добавляет выбранную в библиотеке пачку.

### Случайная выдача

- Используется в слотах **Random по весам**, а также после окончания жёсткой очереди.
- **Вес** задаёт относительную частоту выпадения. Например, вес 20 примерно вдвое вероятнее веса 10.
- Процент справа показывает приблизительную долю пачки среди всех положительных весов.
- Вес `0` исключает пачку из случайной выдачи.

## Палитра

- **+** добавляет цвет; **Изменить** редактирует его ID, название и оттенок.
- ID цвета должен быть уникальным. Это техническое значение, которое используется внутри пачек и JSON.
- **×** удаляет цвет. Если он использовался в пачках, редактор предупредит об изменениях; опустевшие пачки будут удалены.
- Рекомендуется сначала настроить палитру, затем создавать пачки.

## Проверка и передача уровня

Перед экспортом проверьте:

- цель и её количество;
- форму поля и отсутствие случайно активированных ячеек;
- стартовые пачки и замки на поле;
- порядок цветов в каждой пачке;
- первые три позиции жёсткой очереди;
- веса случайной выдачи — должен существовать хотя бы один положительный вес;
- прохождение через **▶ Тестировать**.

Для передачи одного текущего уровня используйте **Скачать JSON**. Для передачи или резервной копии всех сохранённых уровней сначала сохраните изменения, затем используйте **Скачать ZIP**.
